# 工具与数据来源

> 记录用户实际使用的外部系统，以及 agent 如何读写它们。本地 Markdown 是默认低门槛方案，不是唯一方案。

## 待办事项

- 当前工具：本地 Markdown / 滴答清单 / Todoist / Things / Linear / 其他
- 读取内容：任务、截止、项目、优先级、标签
- 写入规则：确认后再写入；不自动创建大量任务

## 日历

- 当前工具：本地 Markdown / Google Calendar / Apple Calendar / Outlook
- 读取内容：固定承诺、会议、出行
- 写入规则：只有用户确认时间块后再写入

## 愿景与知识库

- 当前工具：本地 Markdown / Notion / Obsidian / Logseq
- 读取内容：人生罗盘、长期目标、复盘、笔记
- 写入规则：重要更新先总结给用户确认

## 习惯追踪

- 当前工具：本地 Markdown / 习惯类 App / 打卡表
- 读取内容：习惯完成情况、连续天数、失败记录
- 写入规则：用户确认后再更新；不把单次中断定性为失败

## 复盘与日志

- 当前工具：本地 Markdown / 日记 App
- 读取内容：周/月/项目复盘、情绪日志
- 写入规则：用户明确要求或共同确认后再写入

## MCP 接入原则

1. **读写分离**：读取可以更主动，写入必须更谨慎。
2. **确认后写入**：创建、修改、删除外部事项前先让用户确认。
3. **工具失败可降级**：MCP 不可用时，先输出 Markdown 草案，不阻塞教练过程。
