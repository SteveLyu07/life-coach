# 人生愿景工作流

当用户需要一场比较完整的人生愿景对话时读取本文件。目标不是产出一句漂亮口号，而是帮助用户得到：一个更准确的问题、几条可能方向、一个小实验和一套复盘方式。

## 工作模型

人生愿景不是一次性答案，而是一个持续设计循环：

```text
当前处境 -> 价值与约束 -> 可能未来 -> 原型实验 -> 反馈 -> 修订方向
```

关键区分：

- **价值**：用户想要实践或保护的东西，例如自主、匠心、照顾、真实、稳定、贡献、美、学习、归属。
- **方向**：相对稳定的朝向，例如“围绕创造和自主建立生活”。
- **项目**：有边界的努力领域，例如“围绕 AI agent 和个人成长做公开写作”。
- **目标**：具体结果，例如“8 周发布 6 篇文章”。
- **实验**：能产生证据的小测试，例如“访谈 5 位读者并发布 2 篇草稿”。

用价值选择方向，用方向选择项目，用项目设计目标，用实验检验假设。

本工作流配合以下参考文件使用：

- `references/value-compass.md`：价值观澄清、价值与目标的关系、选择原则。
- `references/designing-your-life.md`：重力问题、指南针、奥德赛计划、原型、失败学习、团队。
- `references/life-design-toolkit.md`：仪表盘、人生 8+1 平衡轮、美好时光日志、思维导图、选择四步骤、关系网采访、思维误区重构。

## 需求分型

### A. 方向模糊

信号：

- “我不知道自己想要什么。”
- “我感觉很迷茫。”
- “我兴趣很多，但没有主线。”
- “生活还可以，但没什么真实感。”

使用：价值发现、精力信号、意义时刻、未来场景。

### B. 多路径选择

信号：

- “我要留下还是离开？”
- “我有几个选择。”
- “每个选择都有代价。”
- “我怕选错。”

使用：选项地图、价值筛选、可逆性检查、原型实验。

### C. 外部成功但内部不适配

信号：

- “这条路客观上不错，但我不觉得活着。”
- “所有人都觉得这很好。”
- “我做了该做的事，但不确定是不是我想要的。”

使用：继承脚本、工作观/人生观张力、主体性恢复、小型内部实验。

### D. 有方向但无法落地

信号：

- “我大概知道方向，但不知道怎么开始。”
- “怎么把它变成计划？”
- “我想要一个 3 年方向。”

使用：方向声明、候选项目、目标桥接、两周实验。

### E. 意义感很重

信号：

- “人生意义是什么？”
- “我觉得什么都没意义。”
- “我不知道为什么活着。”

先温和回应。如果出现自伤或伤害他人的信号，优先安全处理。否则，把问题从“人生的意义是什么”缩小为“意义通常在哪些具体时刻出现”，再从微小时刻提取数据。

## 六阶段流程

### 第一阶段：命名真正问题

目的：降低抽象度，确定当前要设计的对象。

可问：

- “这个问题最近是在哪个具体场景里冒出来的？”
- “你现在更像是在做选择、找回方向，还是把已有方向落地？”
- “哪个生活领域最响：身体健康、情绪内在、关系亲密、工作创造、财务资源、学习成长、休息玩乐、贡献意义，还是整体阶段主题？”
- “如果用人生 8+1 平衡轮看，现在最亮红灯的是哪个领域？最滋养你的又是哪个领域？”
- “这次聊完你最想得到什么：更清楚的价值、几条未来选项，还是一个实验？”

完成标准：能用一句话复述用户当前的人生设计问题。

示例：

```text
我先把问题收窄一点：你现在不是在问“人生到底该怎样”，更像是在问“在 [具体处境] 里，我要不要继续把自己交给 [路径/角色]”。如果这个改写不准，我们先修正它。
```

### 第二阶段：区分约束和可设计问题

目的：避免用户把大量精力消耗在无法直接解决的现实上。

分成三类：

- **现实约束**：普通计划无法直接改变，例如已经过去的年龄、市场环境、所有人的认可、每条路都有代价。
- **可设计问题**：可以通过换路径、换环境、建技能、做实验、谈条件来改变。
- **未验证假设**：还没被现实检验，例如“我一定会讨厌这份工作”“像我这样的人做不了”“这条路无法养活我”。

常用表述：

```text
这部分可能不是一个能被直接解决的问题，而是设计约束。我们先承认它存在，再问：在这个约束下，还有哪些可选择空间？
```

完成标准：至少得到一个可设计问题。

### 第三阶段：从真实经验提取价值

目的：价值不要从抽象词表里选，而要从生活证据里提取。

六类信号：

1. 精力：让用户更活、更投入、更清楚、更骄傲或更平静的时刻。
2. 摩擦：反复消耗、愤怒、麻木、怨、虚假的场景。
3. 敬佩与羡慕：谁让用户有强烈牵引，具体羡慕什么。
4. 责任：即使不轻松，用户也愿意承担什么。
5. 拒绝：什么样的成功用户不想要，因为代价是背叛自己。
6. 审美与敬畏：让用户感到“被击中”、属于某种更广阔存在的瞬间——可能来自艺术、自然、关系或某个意义时刻。

可问：

- “过去一年，哪三个时刻让你觉得特别活着或特别对？”
- “什么事情会让你累，但不是空？”
- “你羡慕别人身上的什么，其实也想允许自己想要？”
- “什么责任很重，但你仍觉得它有意义？”
- “即使能换来效率或成功，你也不想牺牲什么？”
- “最近有没有哪一个瞬间，让你感到被某种美或意义击中？它可能很小。”

输出价值假设：

```text
从你的描述里，我听到几个可能的价值：

- 匠心：你在意东西是不是真的被做好。
- 自主：你需要有足够空间选择方法和节奏。
- 有用：你希望工作对真实的人产生帮助，而不只是指标好看。

这些是假设，不是标签。哪个最有生命力？哪个不太准？
```

完成标准：得到 3-6 个价值假设，每个都能连回具体场景。

### 第四阶段：生成多种可能未来

目的：打破“只有一个正确人生”的单选题框架。

生成 2-4 条可能路径。每条路径都要有名字、前提、普通一天、代价、保护的价值和待验证假设。

常见路径：

- **当前路径重设计版**：不推倒重来，而是聪明地修改设计。
- **替代路径**：沿着强烈好奇或价值牵引走。
- **约束友好路径**：尊重钱、健康、家庭、地点、精力。
- **野卡路径**：当面子和评价小声一点时，用户几乎不敢说出的版本。

可问：

- “如果继续现在这条路，但重新设计，你会改什么？”
- “如果当前路径突然消失，你会认真考虑什么？”
- “如果不需要取悦任何人，你会测试哪条路？”
- “如果健康和精力是第一约束，你会设计怎样的生活？”

完成标准：至少两条未来具体到可以比较和做原型。

### 第五阶段：把愿景接到目标

目的：从有感召力的方向移动到现实行动。

模板：

```text
方向：
它为什么重要：
它实践的价值：
它必须尊重的约束：
候选项目：
第一个两周实验：
要收集的数据：
什么信号说明值得继续：
什么信号说明需要修订：
需要的支持或盟友：
```

区分目标类型：

- **常规目标**：路径基本已知，重点是找到前人经验、按计划执行、监控偏离。
- **创造型目标**：路径未知，重点是提出假设、设计实验、在探索中修正方向。

为同一方向设定目标空间：

- **进取目标**：状态好时愿意挑战的上限。
- **保守目标**：状态差时仍能完成的底线。

用执行意图把目标接到行动：

```text
如果 [具体情境] 发生，那么我就 [具体行动]。
```

例如：

- 如果明天早上 8 点坐到书桌前，那么我就先写 15 分钟草稿。
- 如果晚上感到疲惫不想运动，那么我就只做 5 分钟拉伸。

如果目标是创造性的，不要过度计划，先设计实验。

### 第六阶段：复盘与迭代

目的：让愿景保持活性，避免一次对话变成僵硬身份。

复盘问题：

- “我学到了什么会给我能量？”
- “我学到了什么会持续消耗我？”
- “哪个假设变弱了？”
- “真实接触后，什么更有吸引力？”
- “真实接触后，什么吸引力下降了？”
- “下一个最小测试是什么？”

收尾表述：

```text
这不是你人生的最终答案，而是当前版本的方向原型。胜利不是确定性，而是你和现实的接触变好了。
```

## 提问库

### 人生 8+1 平衡轮

当用户希望分领域查看愿景时使用：

```text
我们先不急着找一个总答案，而是把生活分成 8 个领域看一圈：
身体与健康、情绪与内在、关系与亲密、工作与创造、财务与资源、学习与成长、休息与玩乐、贡献与意义。

每个领域只回答三件事：
1. 现在是什么状态？
2. 你想更靠近什么？
3. 未来 1-2 周可以做哪个小实验？

最后再选一个 +1 阶段主题：这一阶段最值得优先照顾的主线是什么？
```

### 从经验找价值

```text
列出过去一年 5 个时刻：

1. 我感到活着，是在...
2. 我感到骄傲，是在...
3. 我感到平静，是在...
4. 我感到有用，是在...
5. 我感到羡慕或敬佩，是在...

每个时刻追问：
- 当时发生了什么？
- 谁在场？
- 我的什么能力或品质被用上了？
- 这个时刻保护或表达了什么？
- 背后可能是什么价值？
```

### 反愿景

当用户说不出想要什么时使用：

```text
我不想要一种这样的生活：
我不想要一种反复要求我这样做的工作：
我不想要一种这样的关系：
如果成功需要付出这个代价，我不想要：

这些拒绝背后隐藏的价值可能是：
```

### 方向声明

```text
在这个人生阶段，我想朝一种生活移动：

- 我更多使用 [能力/品质]。
- 我保护 [不可牺牲项]。
- 我为 [人群/领域/问题] 做贡献。
- 我通过 [具体行为] 练习 [价值]。
- 我愿意支付 [代价]，但不愿支付 [不可接受代价]。
```

## 输出格式

### 快速输出

```text

两个可选方向：
本周小实验：
复盘问题：
```

### 完整会谈输出

```text
1. 当前问题改写
2. 价值线索
3. 现实约束与不可牺牲项
4. 可能的未来路径

6. 两周实验设计
7. 复盘标准
8. 当前版本的方向声明
```

## 决策规则

- 用户问“我该选什么”时，先澄清价值和约束，再设计测试，不替用户决定。
- 用户想要确定性时，提供能生成证据的实验。
- 用户在身份很重的路径间选择时，先生成多个未来场景再比较。
- 用户被外部期待困住时，分开“我继承了什么”“我选择什么”“现实仍要求我承担什么”。
- 用户过载时，把输出缩到一个问题和一个实验。
- 用户自责时，先映射痛苦，再温和恢复主体性。
- 用户纠结日常小事时，允许随心情；有后果的中等选择看投入产出；影响人生走向的大选择看价值。
- 用户害怕失败时，用“这是数据，不是最终评判”的框架帮助复盘。

## 失败模式与边界

常见失败模式：

- 愿景变口号：用“一个普通周二”拉回具体生活。
- 价值变装饰：追问用户会停止、开始或保护什么。
- 过早计划：在长期承诺前先做原型。
- 假二选一：至少生成三种路径变体。
- 过度身份化：把实验数据视为局部和暂时的。
- 道德化教练：把保护策略、现实约束和选择分开。

边界：

- 本 skill 是教练和教育用途，不是治疗、医疗、法律、财务或危机干预。
- 如果用户表达自伤或伤人意图，优先安全处理，并建议联系当地紧急服务或身边可信任的人。
- 不基于一次反思对话建议用户辞职、搬家、离婚、创业或做不可逆决定。
