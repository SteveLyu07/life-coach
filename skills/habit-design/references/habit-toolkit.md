## 1. 行为方程式检查表

### 核心公式

```text
行为 = 动机 + 能力 + 提示
```

行为发生需要三者同时具备。检查表用于快速诊断哪一个要素缺失。

| 维度 | 检查问题 | 当前状态（1-10） |
|---|---|---|
| 动机 | 我现在有多想做这件事？ |  |
| 能力 | 以最低精力状态，这件事是否足够容易？ |  |
| 提示 | 有没有一个稳定、具体的线索会提醒我？ |  |

### 诊断规则

- **动机低于 5**：连接到更强的价值，或换一个自己更想做的行为。
- **能力低于 5**：使用“能力链”检查并降低阻力。
- **提示缺失**：设计一个行动提示（锚点），而不是依赖记忆或闹钟。

### 能力链检查

```text
□ 时间：是否有足够时间？
□ 资金：是否需要花钱？
□ 体力：是否需要身体能量？
□ 脑力：是否需要思考、决策？
□ 日程：是否与现有日程冲突？
```

最薄弱的环节决定行为是否容易执行。

## 2. 锚点设计法

### 锚点公式

```text
在我 [旧习惯] 之后，我会 [新习惯]。
```

### 选择锚点的标准

- **稳定**：每天发生，时间和情境大致固定。
- **具体**：是一个动作，而不是一个时间段。
- **匹配**：与新习惯在地点、能量水平或主题上相配。
- **频率一致**：锚点和新习惯的发生频率相近。

### 提示类型优先级

| 类型 | 可靠性 | 示例 | 适用场景 |
|---|---|---|---|
| 行动提示（锚点） | 高 | 刷完牙、冲咖啡 | 日常习惯 |
| 情境提示 | 中 | 闹钟、便利贴 | 单次行为、提醒 |
| 人物提示 | 低 | 靠记忆、身体本能 | 饥饿、困倦等生理信号 |

### 常见锚点池

```text
早晨：起床、下床、刷牙、洗脸、冲咖啡、坐到桌前
工作：打开电脑、合上电脑、午休开始、午休结束
晚间：到家、放下包、吃晚饭、洗碗、躺下、关灯
```

## 3. 最小版本原则

### 核心问题

```text
如果今天状态最差，这个行为可以缩小到什么地步？
```

### 两种缩小方式

| 方式 | 说明 | 示例 |
|---|---|---|
| **入门步骤** | 只完成最开始的一个动作 | 想运动 → 换上运动鞋 |
| **缩小规模** | 把理想行为按比例缩小 | 想整理房间 → 收拾一个抽屉 |

### 最小版本示例

| 目标习惯 | 普通版本 | 最小版本 |
|---|---|---|
| 运动 | 跑步 30 分钟 | 换上运动服 |
| 冥想 | 冥想 15 分钟 | 深呼吸 3 次 |
| 阅读 | 读一章 | 读一页 |
| 写作 | 写 1000 字 | 写一句话 |
| 喝水 | 喝 8 杯 | 喝一口 |

### 使用规则

- 最小版本是底线，不是目标。
- 完成最小版本后，可以自然扩展，但不要强迫。
- 如果连续中断，检查最小版本是否还是太大。

## 4. 环境设计技术

### 核心原则

让正确行为更容易，让错误行为更困难。

### 好习惯：减少阻力

- **减少步骤**：把需要的物品放在显眼位置。
- **提前准备**：前一天晚上准备好第二天需要的东西。
- **消除决策**：让习惯变成默认选项。
- **增加提示**：用视觉线索提醒自己。

### 坏习惯：增加阻力

- **物理隔离**：把诱惑物放到需要努力才能拿到的地方。
- **增加步骤**：让坏习惯多一步才能开始。
- **移除提示**：关闭通知、把零食收起来、把手机放到另一个房间。

### 环境设计示例

| 目标 | 环境改变 |
|---|---|
| 多喝水 | 在桌上放一个大水杯 |
| 少刷手机 | 睡前把手机放到另一个房间 |
| 多运动 | 睡前把运动服放在床边 |
| 早睡觉 | 晚上 10 点自动调暗灯光 |
| 多读书 | 在床头放一本书，把手机放到抽屉 |
| 健康饮食 | 把水果放在桌面，零食收进柜子 |

## 5. 奖励与庆祝设计

### 核心原则

行为发生后立刻创造积极情绪，大脑才会记住“这个行为值得重复”。

### 庆祝方式示例

- 说一句“我做到了”或“我很棒”。
- 握拳、微笑、深呼吸。
- 在习惯追踪表上画勾。
- 想象自己身上有光环。

### 庆祝原则

```text
及时：行为完成后 1-3 秒内进行。
简单：不超过几秒，不增加负担。
自然：选择让你真正感到开心的方式。
```

### 奖励与身份一致

奖励不应与目标身份冲突。例如：

- 健身的奖励是按摩或休息，而不是吃冰激凌。
- 存钱的奖励是一次免费活动，而不是冲动购物。

## 6. 习惯叠加

### 公式

```text
继 [当前习惯] 之后，我将 [新习惯]。
```

### 示例

```text
早上起床后：
1. 我会喝一口水。
2. 然后我会做 3 次深呼吸。
3. 然后我会写下今天最重要的 1 件事。
```

### 使用规则

- 一次只加一个新习惯。
- 前一个习惯稳定后，再叠加下一个。
- 链条不宜过长，通常 2-4 个为宜。

## 7. 两分钟规则与诱惑捆绑

### 两分钟规则

新习惯的开始阶段不应超过两分钟。目标是先建立“出现”和“重复”。

示例：

- 想阅读 → 读一页。
- 想运动 → 穿上运动鞋。
- 想写作 → 写一句话。

### 诱惑捆绑

把需要做的事和想要做的事配对：先做需要的事，再做想做的事。

示例：

- 只能在跑步机上听自己喜欢的播客。
- 只能在泡好茶后刷短视频。

## 8. 坏习惯替换

### 替换步骤

1. **识别习惯回路**：触发线索 → 行为 → 即时奖励。
2. **理解需求**：这个习惯满足什么？缓解压力？逃避无聊？获得连接？
3. **设计替代行为**：能满足同样需求，但长期代价更小的行为。
4. **改变环境**：减少旧习惯的提示和便利性。
5. **强化新行为**：给替代行为设置即时奖励或庆祝。

### 示例

| 旧习惯 | 触发线索 | 需求 | 替代行为 | 环境改变 |
|---|---|---|---|---|
| 焦虑时刷短视频 | 感到焦虑 | 快速转移注意力 | 做 5 次深呼吸 | 把短视频 App 放到文件夹深处 |
| 下午吃零食 | 工作疲惫 | 补充能量、休息 | 起身走动 2 分钟，喝水 | 把零食收起来，水果放桌上 |
| 睡前刷手机 | 躺在床上 | 放松、助眠 | 读一页纸质书 | 手机放到客厅充电 |

## 9. 渴求冲浪

### 步骤

1. **注意**：觉察到渴求出现，命名它。
2. **停顿**：不立即行动，给自己 60 秒。
3. **观察**：注意身体感觉、情绪、想法的变化。
4. **呼吸**：用缓慢呼吸陪伴渴求。
5. **选择**：等波浪过去后，再决定如何回应。

### 关键提醒

渴求冲浪不是用来“抵抗”，而是用来“观察”。越抵抗，冲动越强；越观察，冲动越容易过去。

## 10. 中断恢复协议

### 核心信念

中断是习惯的一部分，不是习惯的结束。

### 恢复步骤

1. **标记**：承认中断发生了，不夸大。
2. **温和**：不自我攻击，把中断当作数据。
3. **分析**：是什么情境导致中断？提示失效？能力门槛太高？精力不足？
4. **缩小**：回到最小版本，重新开始。
5. **重建**：尽快连续两次完成，恢复连续性。

### Never Miss Twice 规则

- 允许错过一次。
- 但不要连续错过两次。
- 错过一次后，下一次要尽可能完成，即使只是最小版本。

### 中断后不该做的事

- 加倍补偿。
- 全面自我否定。
- 彻底放弃。

## 11. 习惯追踪与复盘

### 追踪原则

- **简单**：追踪方式越简单，越容易坚持。
- **可见**：把追踪记录放在显眼位置。
- **即时**：完成后立刻记录。
- **友善**：追踪是为了支持，不是评判。

### 追踪方式

| 方式 | 示例 |
|---|---|
| 纸质日历 | 完成后打一个勾 |
| Habit tracker App | 记录每日完成情况 |
| 习惯伙伴 | 每天互相汇报 |
| 简单清单 | 每天睡前勾选 |

### 问责伙伴

找一个能监督你的人，让坏习惯立即产生社交后果。问责伙伴可以是朋友、家人或同事。

### 复盘问题

每周问自己：

```text
1. 这周我完成了几天？
2. 中断通常发生在什么情境？
3. 最小版本是否还是太大？
4. 锚点是否稳定？
5. 奖励是否让我想继续？
6. 环境设计是否有效？
7. 下周我要调整什么？
```

### 调整方向

| 问题 | 调整方向 |
|---|---|
| 总是忘记 | 强化提示，换更稳定的锚点 |
| 启动困难 | 进一步缩小最小版本，减少环境阻力 |
| 做了但没动力 | 调整奖励，连接更强价值 |
| 中断后回不来 | 降低重启门槛，预设恢复规则 |
| 觉得无聊 | 增加变化或社交元素；稍微提高难度 |

## 12. 脚本示例

### 脚本 1：帮助用户把愿望变成行为

```text
用户：我想变得更健康。

教练：听起来健康对你很重要。如果“更健康”变成每天可以做的一件小事，
你觉得那会是什么？比如，早上喝一杯水、睡前拉伸 2 分钟、或者每天多吃一份蔬菜？
```

### 脚本 2：帮助用户找到锚点

```text
用户：我想每天冥想，但总是忘。

教练：你每天一定会做、而且不会忘的一件事是什么？比如刷牙、冲咖啡、或者坐到办公桌前？
我们可以把冥想绑定在那件事之后，哪怕只是深呼吸 3 次。
```

### 脚本 3：帮助用户缩小行为

```text
用户：我想每天运动 30 分钟。

教练：30 分钟很好。但如果今天状态最差、只能做一点点，你觉得最小不能放弃的版本是什么？
比如换上运动服、做一个俯卧撑、或者原地走 1 分钟？
```

### 脚本 4：帮助用户替换坏习惯

```text
用户：我一焦虑就刷短视频，刷完更焦虑。

教练：刷短视频在那一刻给你什么？是转移注意力、放松、还是逃避？
我们可以找一个也能带来类似感受，但不会有后续焦虑的行为，比如深呼吸 5 次、听一首歌、或者起身走两步。
```

### 脚本 5：帮助用户从中断中恢复

```text
用户：我坚持了两周，昨天断了，感觉前功尽弃。

教练：中断是习惯的一部分，不是终点。很多习惯的建立过程中都会中断。
重要的是“never miss twice”——允许一次中断，但不让它连续发生。
今天我们只回到最小版本，怎么样？
```
