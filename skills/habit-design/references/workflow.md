## 工作模型

习惯的形成和维持取决于一个简单方程式：

```text
行为 = 动机 + 能力 + 提示
```

当三者同时出现时，行为就会发生。习惯设计的核心是：

- 提高动机（让它有意义）
- 降低能力门槛（让它容易做）
- 设置可靠提示（让它被记得）
- 设计即时奖励（让它想重复）

关键区分：

- **愿望**：抽象结果，如“我想更健康”。
- **行为**：具体可观察的动作，如“每天早上喝一杯水”。
- **最小版本**：在最低动机、最低精力时也能完成的版本，如“喝一口水”。
- **锚点**：已有的稳定习惯或情境，新行为将绑定在它之后发生。
- **提示**：触发行为的线索，可以是时间、地点、前一个动作、情绪或社交场景。
- **奖励**：行为发生后立刻获得的正向反馈，让大脑记住这个行为值得重复。
- **环境设计**：通过改变物理或数字环境来减少阻力、增加提示。

好习惯设计的核心不是“更努力”，而是“让正确的事更容易发生”。

### 双重视角：B=MAP 与四定律

本工作流整合两个公开框架，用于不同阶段：

### 福格行为模型 B=MAP

用于**诊断行为为什么没有发生**以及**设计最小可行行为**：

```text
行为 = 动机 + 能力 + 提示
```

当行为没有发生时，检查：

- 动机是否足够，或是否需要绕过动机波动。
- 能力链（时间、资金、体力、脑力、日程）哪个环节最薄弱。
- 提示是否可靠，是否使用行动提示（锚点）。

### 掌控习惯四定律

用于**设计好习惯和破除坏习惯**的对称策略：

| 培养好习惯 | 破除坏习惯 |
|---|---|
| 让它显而易见 | 让它脱离视线 |
| 让它有吸引力 | 让它缺乏吸引力 |
| 让它简便易行 | 让它难以施行 |
| 让它令人愉悦 | 让它令人厌恶 |

七阶段流程的每一步都可以映射到四定律中的某一条或多条。

## 需求分型

### A. 建立新习惯

信号：

- “我想每天冥想。”
- “我想开始运动。”
- “我想养成早睡习惯。”

使用：愿望→行为转换、锚点设计、最小版本、环境支持、即时奖励。

### B. 替换坏习惯

信号：

- “我想戒掉睡前刷手机。”
- “我想用喝水替代吃零食。”
- “我想改掉一焦虑就刷短视频的习惯。”

使用：习惯回路分析、替代行为设计、环境重塑、渴求冲浪、奖励替换。

### C. 恢复已中断的习惯

信号：

- “我之前每天运动，断了两周，现在动不起来了。”
- “习惯养到一半又丢了，怎么回来？”

使用：中断恢复规则、最小重启版本、自我友善、不苛责回归。

### D. 降低习惯摩擦

信号：

- “我知道该做，但总是因为准备麻烦而放弃。”
- “习惯太难启动。”

使用：能力分析、环境设计、减少步骤、提前准备、降低决策点。

### E. 让习惯长期可持续

信号：

- “我坚持过 21 天，但后面又停了。”
- “怎么让习惯变成像刷牙一样自然？”

使用：习惯叠加、自动化设计、记录追踪、定期复盘、弹性设计。

## 七阶段流程

### 第一阶段：明确愿望与目标行为

目的：把抽象愿望变成具体、可观察的行为。

可问：

- “你想养成的习惯，最终想带来什么改变？”
- “如果愿望实现了，你具体会做什么不一样的事？”
- “这个习惯每天至少出现一次的具体动作是什么？”

示例转换：

- 愿望“我想更健康” → 行为“每天早上喝一杯水”。
- 愿望“我想多读书” → 行为“每天睡前读一页书”。

完成标准：能用一句话说出具体行为，包含“谁在什么时间、什么地点、做什么”。

### 第二阶段：分析行为方程式

目的：检查动机、能力、提示是否都具备。

可问：

- “这个行为对你为什么重要？它连接到什么价值？”
- “以你现在的精力和时间，这个行为容易做到吗？”
- “有什么能提醒你开始这个行为的线索吗？”

分析模板：

```text
动机（1-10）：
能力（1-10）：
提示（有/无/弱）：
```

完成标准：清楚行为发生的三个要素现状。

### 第三阶段：找到锚点线索

目的：把新行为绑定到已有的稳定习惯或情境上。

可问：

- “你每天一定会做的一件事是什么？”
- “这个新行为可以放在哪件事之后？”
- “什么时间/地点你最不可能忘记？”

示例：

- “刷完牙后，做 5 次深呼吸。”
- “坐到办公桌前，先写下今天最重要的 1 件事。”
- “晚上关上电脑，就把手机放到客厅充电。”

完成标准：有一个具体、稳定的锚点。

### 第四阶段：缩小到最小版本

目的：确保即使在坏日子也能完成，从而保持连续性。

可问：

- “如果今天状态最差，这个行为可以缩小到什么地步？”
- “有没有一个 30 秒版本？”
- “能不能先只做一次？”

示例：

- 运动 → 换上运动服。
- 冥想 → 深呼吸 3 次。
- 写作 → 写一句话。
- 整理 → 收拾一个抽屉。

完成标准：得到一个“不可能失败”的最小版本。

### 第五阶段：设计环境支持

目的：让正确行为更容易，让错误行为更困难。

可问：

- “要开始做这件事，你需要准备什么？”
- “环境中有什么在阻碍你？”
- “你能提前把工具放在哪里？”

环境设计示例：

- 想多喝水：在桌上放一个装满水的水杯。
- 想少刷手机：睡前把手机放到另一个房间。
- 想多运动：睡前把运动服放在床边。
- 想早起：把闹钟放在必须起床才能关掉的地方。

完成标准：环境已经为新行为减少至少一个阻力或增加一个提示。

### 第六阶段：设计即时奖励

目的：让大脑在行为发生后立刻获得正向反馈。

可问：

- “做完这个行为后，你可以立刻给自己什么小奖励？”
- “这个行为本身有没有让你觉得好的部分？”
- “怎么让完成习惯的感觉更明显？”

奖励示例：

- 打一个勾、画一个笑脸。
- 说一句鼓励自己的话。
- 记录到成就清单。
- 给自己 2 分钟做喜欢的事。

完成标准：行为完成后有明确的即时奖励。

### 第七阶段：预设中断恢复与复盘

目的：让中断成为正常数据，而不是习惯结束的信号。

规则：

- 中断必然发生，关键是从中断中恢复。
- 用“never miss twice”原则：允许中断一次，但尽快连续两次。
- 中断后不 doubling down，不报复性补做，只回到最小版本。

复盘问题：

- 这个习惯连续坚持了几天？
- 中断通常发生在什么情境？
- 最小版本是否还是太大？
- 锚点是否稳定？
- 奖励是否有效？

收尾表述：

```text
习惯不是一条直线，而是螺旋上升的。中断不代表失败，只代表设计还需要微调。
```


## 决策规则

- 用户只说“我想养成习惯”时，先问具体愿望和行为，不直接给模板。
- 用户目标行为太大时，坚持缩小到最小版本。
- 用户找不到锚点时，从“每天一定会做的事”入手。
- 用户动机不足时，先连接到价值，而不是靠意志力硬推。
- 用户反复中断时，检查最小版本是否还是太大，或锚点是否不稳定。
- 用户想戒掉坏习惯时，优先设计替代行为和环境重塑，而不是靠压抑。
- 用户自责时，先温和地回到“习惯是设计问题，不是人格问题”。

## 失败模式与边界

常见失败模式：

- **愿望太抽象**：只停留在“我想更健康”，没有具体行为。
- **最小版本还是太大**：用户说“每天 5 分钟”，但坏日子连 5 分钟也做不到。
- **锚点不稳定**：把习惯绑定到一个本身不规律的事件上。
- **忽视环境**：只靠决心，不改变环境阻力。
- **奖励延迟**：行为发生后没有即时反馈。
- **中断后自我攻击**：把一次中断当作习惯失败，彻底放弃。
- **同时改变太多**：一次建立多个习惯，资源分散。

边界：

- 本 skill 是教练和教育用途，不是治疗、医疗、法律或财务建议。
- 对于成瘾性行为（如物质依赖、严重暴食/厌食、病理性赌博等），建议寻求专业支持。
- 不替用户做重大人生决策或不可逆承诺。
