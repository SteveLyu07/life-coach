# 复盘案例库

这些是合成的公开案例，用作回应模式。不要把它们说成真实来访者故事。

## 案例 1：周复盘只看到自己的失败

用户：

```text
这周又废了。计划每天运动，只去了两次；想早睡，结果天天熬夜；工作也没完成。我感觉自己一点自控力都没有。
```

回应思路：

先不被用户的自我评价带走，而是帮助ta重建事实，并强制找到有效做法。用“成就清单”打破只关注失败的惯性，然后只选 1–2 个变量做下一轮调整。

推荐话术：

```text
你用了三个“又”“也”“都”来形容这周，这个框架下很难找到出路。我们先做一件事：列出这周你做对了什么，哪怕很小。然后我们只选一个最想调整的变量，而不是同时改运动、睡眠、工作三件事。
```

下一步：

- 列出 3 件这周做对或推进的事。
- 分析熬夜的具体情境：是拖延、愉悦感、还是白天自主感不足？
- 只选一个变量调整，例如：睡前半小时把手机放到客厅。
- 观察一周，看睡眠变化。

常见失败模式：

- 用户找不到任何“做对的事”，需要更温和地引导。
- 用户想同时改多个变量，需要坚持只选一个。
- 用户把一次周复盘扩大成“我的人生就是这样”。

## 案例 2：项目失败后过度自责

用户：

```text
我负责的项目延期了，老板很不满意。我觉得都是我的错，如果我当时更努力一点就不会这样。
```

回应思路：

帮助用户从“全怪我”转向“分离变量”。区分可控与不可控因素，看到外部依赖、资源限制、沟通问题等。然后用复盘四步法，先看做对了什么，再分析偏离。

推荐话术：

```text
项目延期一定有你的一部分责任，但通常不会只有你的责任。我们先不急着下结论，而是把时间线列出来：项目有哪些关键节点？每个节点发生了什么？哪些因素是你能控制的，哪些不是？
```

下一步：

- 按时间线重建项目关键事件。
- 列出可控因素、他人因素、情境因素各占多少比例。
- 识别至少一个你做对的地方。
- 只选一个下一轮会调整的变量，例如：更早暴露风险、增加周报频率。

常见失败模式：

- 用户坚持把所有责任揽到自己身上。
- 用户把责任全推给外部，失去可行动空间。
- 复盘变成对老板的抱怨会。

## 案例 3：不知道新方法对自己有没有用

用户：

```text
我试了两周番茄工作法，感觉一般。有时候有用，有时候完全没用。不知道还要不要继续。
```

回应思路：

用“考官视角”帮助用户把方法当作实验对象，而不是标准答案。通过对比不同条件下的使用效果，找出适合用户的变式。

推荐话术：

```text
番茄工作法不是对每个人都一样有效。你已经有两周数据了，我们可以像考官一样给它打分。哪些天它有用？哪些天没用？当时的任务类型、精力状态、环境有什么不同？
```

下一步：

- 记录两周内使用番茄的具体情况。
- 对比有用和没用的日子的差异。
- 测试变式：长单元、短单元、不同任务类型。
- 根据数据决定是否保留、修改或放弃这个方法。

常见失败模式：

- 用户因为“有时没用”就全盘否定。
- 用户没有记录具体数据，只能凭印象判断。
- 用户把方法当作必须服从的规则，产生抗拒。

## 案例 4：反复卡在同一个模式

用户：

```text
我发现自己每次都是开始时热情满满，做一周就放弃。减肥、学吉他、写日记，全都是这样。我是不是注定坚持不了？
```

回应思路：

这是长期模式复盘。帮助用户看到“高开低走”这个模式，并把它当作了解自己的入口，而不是人格缺陷。用“以问题为师”和“连接意图”探索模式背后的功能。

推荐话术：

```text
“高开低走”不是一个缺陷，它是一个你反复使用的策略。它可能帮你回避了什么，比如失败的风险、无聊的中间阶段、或者别人的期待。我们先不急着改它，而是理解它：这个模式在保护你什么？又在让你付出什么代价？
```

下一步：

- 列出三个“高开低走”经历的共同前提。
- 探索模式的功能：可能是避免失败、追求新鲜感、或回应外部期待。
- 设计一个对照实验：下一次只选一个最小目标，不追求热情，只追求连续性。
- 观察指标：连续天数，而非成果大小。

常见失败模式：

- 用户把模式等同于人格，产生宿命感。
- 用户想一次性改掉所有相关行为。
- 实验目标仍然太大，重复旧模式。

## 案例 5：日复盘变成流水账

用户：

```text
我每天睡前都写日记复盘，但感觉只是在记录今天做了什么，没什么用。
```

回应思路：

日复盘流于流水账，是因为缺少“洞察—调整”环节。帮助用户从记录事实转向提取一个有效做法、一个偏离、一个最小调整。

推荐话术：

```text
记录事实只是复盘的第一步。我们可以给你的日复盘加三个问题：今天我做对了什么？哪里和预期不一样？明天我只改一件事，会改什么？这样日记就从流水账变成了学习工具。
```

下一步：

- 在日复盘模板中加入：有效做法、偏离/卡点、明天最小调整。
- 连续记录一周，重点看“最小调整”是否被执行。
- 每周回顾一次，看哪些调整有效。

常见失败模式：

- 用户加了问题但回答得太笼统。
- 用户写下调整但第二天不做，需要连接到执行意图。
- 用户期望日复盘立刻带来巨大改变。
