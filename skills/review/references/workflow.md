## 工作模型

复盘是一个从经历到学习的循环：

```text
行动 -> 结果 -> 重建事实 -> 分离变量 -> 识别模式 -> 提取洞察 -> 调整假设 -> 再行动
```

关键区分：

- **事实**：客观发生了什么，可观察、可记录。
- **解释**：我们对事实赋予的意义，可能正确也可能错误。
- **情绪**：经历中产生的感受，是信号，但不等于事实。
- **模式**：反复出现的事件序列或反应方式。
- **假设**：我们对“什么会导致什么”的预测，可以被验证或修正。
- **洞察**：从数据中得出的、能指导下一次行动的认识。
- **调整**：下一轮只改一个或少数几个变量。

好复盘不是更深刻地批评自己，而是更准确地理解现实。

## 需求分型

### A. 日/周复盘

信号：

- “这周过得怎么样，我想回顾一下。”
- “每天都忙忙碌碌，但不知道收获了什么。”
- “周末想复盘一下这周。”

使用：成就清单、三件有效做法、偏离记录、下一轮最小调整。

### B. 项目复盘

信号：

- “这个项目结束了，我想总结一下。”
- “项目没达到预期，我想知道原因。”
- “下一个项目怎么避免同样的坑？”

使用：项目时间线、关键决策点、假设验证、变量分离、复盘四步法。

### C. 失败复盘

信号：

- “我搞砸了，想复盘一下。”
- “这次考试/面试/演讲没发挥好。”
- “我又没按计划执行。”

使用：复盘四步法、为偏离辩护、先看做对了什么、区分可控与不可控。

### D. 习惯/实验复盘

信号：

- “这个习惯我练了两周，效果不明显。”
- “我想看看这个实验告诉我什么。”
- “这个方法对我有用吗？”

使用：考官视角、预测误差学习、进度条思维、对照实验。

### E. 长期模式复盘

信号：

- “为什么我总是卡在同一个地方？”
- “这种模式反复出现。”
- “我是不是有什么深层问题？”

使用：模式识别、连接意图、以问题为师、总结支点。

## 七阶段流程

### 第一阶段：设定复盘范围

目的：明确这次复盘的对象和边界，避免变成泛泛而谈。

可问：

- “你想复盘的是一天、一周、一个项目，还是一种反复出现的模式？”
- “这次复盘你最想回答什么问题？”
- “你希望聊完后带走什么：一个洞察、一个调整，还是一种理解？”

完成标准：清楚这次复盘的时间范围、对象和核心问题。

### 第二阶段：重建事实

目的：不带评判地还原发生了什么。

方法：

- 按时间线列出关键事件。
- 记录做了什么、没做什么、结果如何。
- 不加入“为什么”“本应该怎样”等解释。

可问：

- “从 [起点] 到 [终点]，具体发生了什么？”
- “你做了什么？没做什么？”
- “结果是什么？有哪些可观察的数据？”

完成标准：能用几句话客观描述事件经过和结果。

### 第三阶段：分离变量

目的：区分哪些因素影响了结果，避免把所有原因混在一起。

变量类型：

- **行动**：我做了什么、没做什么、怎么做的。
- **情境**：时间、地点、环境、工具、他人是否在场。
- **精力**：睡眠、情绪、身体状态、压力水平。
- **认知**：当时的想法、预期、假设、注意力焦点。
- **外部**：他人行为、突发事件、市场变化、运气。

可问：

- “当时你的状态怎么样？”
- “环境中有哪些因素在支持你或干扰你？”
- “你当时怎么理解这件事的？”
- “有哪些事是你控制不了的？”

完成标准：能把影响因素分类，而不是笼统归因。

### 第四阶段：寻找有效做法

目的：先找到已经有效的东西，建立继续行动的基础。

可问：

- “这次经历中，你做对了什么？”
- “哪些小行动推动了进度？”
- “有哪些内在优势或外部资源帮到了你？”
- “如果复制这些条件，会不会更容易成功？”

完成标准：至少识别出 1–3 个值得保留或放大的有效做法。

### 第五阶段：分析偏离与卡点

目的：理解哪里没按预期发生，以及为什么。

可问：

- “哪里和你预期的不一样？”
- “这个偏离是计划问题、执行问题、情境问题，还是假设问题？”
- “如果重来一次，哪个变量最可能改变结果？”
- “这个卡点是一次性的，还是反复出现的？”

完成标准：清楚偏离的类型和可能原因。

### 第六阶段：提取模式

目的：从单次经历中看到更长期的规律。

可问：

- “这种情况以前发生过吗？”
- “每次发生时，有什么共同的前提？”
- “你的反应方式是不是也在重复？”
- “这个模式在保护你什么？又在让你付出什么代价？”

完成标准：能描述一个可能的重复模式，或判断这是一次性事件。

### 第七阶段：制定下一轮调整

目的：把洞察变成可执行的小改变。

原则：

- 一次只改 1–3 个变量。
- 改变要足够小，能被测试。
- 明确观察指标：什么信号说明调整有效？

可问：

- “如果下一轮只改一件事，你会改什么？”
- “这个调整具体会改变哪个行动或情境？”
- “你用什么指标来判断它是否有效？”

收尾表述：

```text
这不是对过去的最终审判，而是为下一轮行动收集的情报。我们带着一个更准确的假设，再试一次。
```

## 输出格式

### 快速输出

```text
复盘范围：

一个有效做法：
一个偏离/卡点：
一个可能模式：
下一轮最小调整：
观察指标：
```

### 完整会谈输出

```text
1. 复盘范围与核心问题
2. 事实重建
3. 变量分离
4. 有效做法
5. 偏离与卡点分析
6. 模式识别
7. 下一轮调整与观察指标
```

## 决策规则

- 用户只说“我想复盘一下”时，先问范围和最想回答的问题。
- 用户一上来就自责时，先温和地把焦点转向“发生了什么”。
- 用户过度分析原因时，拉回到“哪些变量可以改变”。
- 用户把失败全归因于自己时，引导区分可控与不可控因素。
- 用户找到很多问题要改时，帮助选出 1–3 个最关键变量。
- 用户复盘后更沮丧时，检查是否忽视了有效做法和自我关怀。
- 用户反复复盘同一模式但不行动时，把焦点转向“最小下一步”。

## 失败模式与边界

常见失败模式：

- **复盘变检讨**：只关注哪里错了，忽视做对的部分。
- **归因过度内倾**：把所有原因都归为自己，忽视情境和外部因素。
- **归因过度外倾**：把所有原因都归于外部，失去可行动空间。
- **改变太多**：一次性想改所有变量，导致下一轮崩溃。
- **分析瘫痪**：反复复盘但不进入行动。
- **后见之明偏误**：用现在的信息批判当时的决定。
- **模式泛化**：从一次事件得出“我总是这样”的笼统结论。

边界：

- 本 skill 是教练和教育用途，不是治疗、医疗、法律或财务建议。
- 不替用户做重大人生决策或不可逆承诺。
- 当用户有严重情绪困扰时，建议寻求专业支持。
