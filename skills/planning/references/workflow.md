## 工作模型

计划是一个循环，而不是一次性的完美蓝图：

```text
澄清意图 -> 设定目标 -> 盘点约束 -> 拆解任务 -> 安排承诺 -> 执行与偏离 -> 复盘与再计划
```

关键区分：

- **意图**：你为什么想做这件事，它连接到什么价值或方向。
- **目标**：可评估的终点，包含截止时间和完成标准。
- **项目**：有边界的努力领域，通常需要多步骤完成。
- **任务**：项目中的具体单元，能在一次专注时段内推进。
- **下一步行动**：足够小到可以立刻启动的第一步。
- **计划**：让目标在真实时间、精力和约束中发生的前瞻性安排。
- **执行意图**：用「如果……那么……」把情境线索和行动提前绑定。

好计划的核心不是填满每一小时，而是降低重要行动的启动阻力，并为变化预留缓冲。

## 需求分型

### A. 日/周安排

信号：

- “今天/这周要做什么？”
- “事情太多，不知道从哪开始。”
- “计划总是被打乱。”

使用：要事优先、执行意图、番茄单元、缓冲设计、日程化。

### B. 目标拆解

信号：

- “我有一个大目标，不知道怎么拆。”
- “目标太大，一直动不了。”
- “怎么把愿景变成任务？”

使用：目标分层、常规型/创造型目标、任务拆分、下一步行动、保守/进取目标空间。

### C. 项目推进

信号：

- “这个项目一直卡着。”
- “依赖太多。”
- “怎么排里程碑？”

使用：关键路径、增量交付、依赖梳理、风险缓冲、复盘点。

### D. 启动困难

信号：

- “我知道该做，但就是开始不了。”
- “一坐下就刷手机。”
- “启动阻力特别大。”

使用：2 分钟规则、5 分钟启动、情境支持、行动激活、降低门槛。

### E. 计划总是失败

信号：

- “我做了计划但执行不了。”
- “每次复盘都在自责。”
- “计划不如变化快。”

使用：复盘四步法、成就清单、进度条思维、偏离作为数据、计划是为变化而生的。

## 七阶段流程

### 第一阶段：澄清意图

目的：确认这个计划到底为什么服务，避免在错误方向上高效率执行。

可问：

- “这件事如果做成了，会给你的生活带来什么变化？”
- “它连接到你真正在意的什么？”
- “这是你选择的，还是你觉得应该做的？”
- “如果这件事不做，最坏会怎样？”

完成标准：能用一句话说出这个计划服务的价值或方向。

### 第二阶段：设定目标

目的：把意图变成可评估、可调整的目标。

目标必须回答：

- **why**：为什么重要？
- **what**：具体要做到什么程度？
- **when**：什么时候完成或检查？

区分目标类型：

- **常规型目标**：路径已知，重点是找前人经验、按计划执行、监控偏离。
- **创造型目标**：路径未知，重点是设定问题和“找到答案的迹象”，用实验探索。

设定目标空间：

- **进取目标**：状态好时愿意挑战的上限。
- **保守目标**：状态差时仍能完成的底线。

完成标准：得到一个具体、可评估、有弹性的目标。

### 第三阶段：盘点约束

目的：让计划建立在真实土壤上，而不是理想日历上。

收集：

- 已有承诺和不可移动事项
- 截止日期和外部依赖
- 自己的精力节律（何时效率高、何时低落）
- 可用支持（人、工具、环境）
- 可能的风险和意外

可问：

- “这一周/一天里，哪些时间已经被占用了？”
- “你精力最好的时段通常在什么时候？”
- “这件事依赖谁？他们需要什么时候给你反馈？”
- “最可能打乱计划的因素是什么？”

完成标准：清楚可用时间、精力窗口和风险点。

### 第四阶段：拆解与匹配

目的：让任务大小与真实执行能力匹配。

拆分原则：

- 每个任务能在一次专注时段内推进。
- 复杂成果先搭骨架，再补内容，再优化。
- 多个 10 分钟小任务可以合并成一个专注单元。
- 大任务拆到不超过 5–7 个专注单元。

识别下一步行动：

- 足够具体，知道第一步做什么。
- 足够小，能在 2 分钟内启动。
- 不依赖其他未决事项。

完成标准：得到一组大小合适、有明确下一步的任务。

### 第五阶段：安排与承诺

目的：让重要事项在真实时间中发生。

方法：

- **要事优先**：每天先保护 1–3 件最重要的事，把最高精力时段留给它们。
- **日程化**：给个人事项安排固定时间，像对待会议一样保护它。
- **执行意图**：用「如果 [情境]，那么 [行动]」固化启动条件。
- **事件线索**：当外部时间结构不足时，用「做完 A 就做 B」的链条组织一天。

完成标准：重要事项已被安排进具体时段或触发条件。

### 第六阶段：执行与偏离

目的：按计划行动，同时把偏离当作正常数据。

执行原则：

- 先开始，再优化。允许“只做 5 分钟”。
- 一次只聚焦一个主线任务。
- 用专注单元（如番茄工作法）组织行动和休息。
- 遇到偏离时，先记录，不立即自我批评。

完成标准：行动已发生，偏离已被记录。

### 第七阶段：复盘与再计划

目的：从这次循环中学习，让下一次计划更准确。

复盘问题：

- 哪些行动推动了目标？
- 哪些行动被证明不重要？
- 时间预估偏了多少？原因是什么？
- 偏离是意外、习惯、情绪还是目标本身的问题？
- 下一个最小调整是什么？

收尾表述：

```text
这个计划不是最终答案，而是当前版本的最优猜测。下一次计划会根据这次的真实数据变得更准。
```

## 提问库

### 澄清意图

```text
这件事如果做成了，会给你的生活带来什么变化？
它连接到你真正在意的什么？
这是你选择的，还是你觉得应该做的？
如果这件事不做，最坏会怎样？
```

### 设定目标

```text
你想在什么时间之前，完成什么？
完成到什么程度，你会知道“成了”？
这件事的 why 是什么？
如果状态好，你愿意挑战的上限是什么？
如果状态差，你能接受的底线是什么？
```

### 盘点约束

```text
这一周/一天里，哪些时间已经被占用了？
你精力最好的时段通常在什么时候？
这件事依赖谁？他们需要什么时候给你反馈？
最可能打乱计划的因素是什么？
```

### 拆解任务

```text
这件事可以分成哪几个中间成果？
第一步最小可以有多小？
哪些小任务可以合并处理？
哪个任务是关键路径上的？
```

### 复盘偏离

```text
实际发生了什么？
这和你的预期差在哪里？
这个偏离告诉了你什么？
下一次计划，你会调整目标、时间、环境还是启动方式？
```

## 输出格式

### 快速输出

```text
我听到的核心需求：
这个计划服务的意图：
一个可评估的目标：
今天/本周最小的一步：
一个执行意图：
复盘问题：
```

### 完整会谈输出

```text
1. 澄清后的意图
2. 目标与目标空间（进取/保守）
3. 现实约束与可用资源
4. 任务拆解与关键路径
5. 安排与执行意图
6. 风险与缓冲
7. 复盘标准
```

## 决策规则

- 用户只说“我要做计划”时，先问计划周期和意图，不直接给模板。
- 用户任务过多时，先帮其选出 1–3 件最重要的事，再排其余。
- 用户反复计划失败时，先复盘上一次计划哪里不准，而不是让ta再做一个更完美的计划。
- 用户启动困难时，优先降低第一步门槛，而不是增强意志力要求。
- 用户面对创造性目标时，强调实验和模糊计划，避免过度拆解。
- 用户面对常规目标时，强调清单、流程和监控偏离。
- 用户自责时，先把偏离数据化，再温和地回到再计划。
- 用户精力低落时，建议切换到保守目标或恢复性活动。

## 失败模式与边界

常见失败模式：

- **计划变愿望清单**：只列想做的事，不考虑时间和精力约束。
- **过度拆解**：把创造性目标拆成不可能准确预估的步骤。
- **没有缓冲**：把每一小时排满，导致一次意外就全盘崩溃。
- **只看终点**：忽视已取得的进度，持续自我否定。
- **把偏离当失败**：没有从偏离中提取数据，进入自责循环。
- **计划与执行脱节**：计划时没有考虑执行情境和启动阻力。
- **忽视意图**：在错误方向上高效率执行。

边界：

- 本 skill 是教练和教育用途，不是治疗、医疗、法律或财务建议。
- 不替用户做重大人生决策或不可逆承诺。
- 当用户有严重焦虑、抑郁或危机信号时，优先建议专业支持。
