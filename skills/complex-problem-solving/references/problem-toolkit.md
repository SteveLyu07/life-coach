## 1. 问题定义公式

### 一句话问题

```text
谁 + 在什么情境下 + 做了什么/没做什么 + 带来了什么影响
```

示例：

- 模糊："工作很不顺。"
- 具体："我连续两周每天工作超过 10 小时，但项目进度仍然落后，让我焦虑和疲惫。"

### 好问题的标准

- 包含具体的人、事、时间、情境。
- 不只是一个概括化评价。
- 有明确的“终点”——解决后是什么样子。

## 2. 问题空间图

### 三要素

```text
起点：问题刚出现时的状态
终点：问题解决后的理想状态
当前位置：现在处于哪里（0-10）
```

### 使用方式

1. 描述起点和终点，越具体越好。
2. 评估当前位置。
3. 下一步目标不是直接到终点，而是往前推一小步。

示例：

```text
起点：每天工作 10 小时以上，项目落后
终点：每天工作 8 小时以内，项目按期交付
当前位置：3 分
下一步（到 4 分）：列出本周必须完成的三项任务
```

## 3. 问题解决决策树

在正式解决前，先回答四个问题：

```text
1. 我愿意解决这个问题吗？
   ├─ 否 → 先回到意愿和价值澄清
   └─ 是 → 进入下一步

2. 这个问题值得解决吗？
   ├─ 是假想敌 / 不可改变的事实 / 可忽略的小事 → 停止或重新框定
   └─ 是真正有价值的问题 → 进入下一步

3. 以我现在能力，能解决吗？
   ├─ 能 → 进入解决流程
   ├─ 需要学习 → 设计学习实验
   └─ 需要求助 → 寻找外部资源

4. 我是在解决问题，还是在评价自己？
   ├─ 评价自己 → 解套，回到目标
   └─ 解决问题 → 继续
```

## 4. 伪问题识别

### 三类伪问题

| 类型 | 判断问题 | 例子 | 应对 |
|---|---|---|---|
| **假想敌** | 我是不是把假设或偏见当成了敌人？ | 认为同事故意针对自己 | 回到事实，停止战斗 |
| **不可改变的事实** | 我是不是想消除本就无法消除的东西？ | 想让负性情绪彻底消失 | 接受、绕过、重新框定目标 |
| **可忽略的小事** | 我是不是想彻底消灭一个影响很小的问题？ | 追求房间绝对整洁 | 止损、低成本减轻、容忍 |

### 快速检查

```text
□ 我想解决的是现实问题，还是想证明自己是对的？
□ 这个问题里有没有不可改变的事实？
□ 不彻底解决这个问题，后果真的严重吗？
□ 解决它的收益是否明显大于成本？
```

## 5. 例外挖掘法

### 核心思想

任何问题都有不发生或影响较小的时刻。找到这些例外，就能发现改变的线索。

### 提问

```text
什么时候这个问题没发生？
什么时候问题的影响比较小？
那时你做了什么不同的事？
那时环境有什么不同？
```

## 6. 内部与外部资源清单

### 内部资源

```text
□ 过去成功解决类似问题的经验
□ 已具备的相关技能
□ 个人的品格优势（耐心、韧性、创造力等）
□ 例外时刻中有效的方法
□ 对自己影响最大的价值观
```

### 外部资源

```text
□ 可以一起行动的盟友
□ 在该问题上更有经验的人
□ 可靠的信息来源
□ 可以购买的专业服务
□ 可以提供情感支持的人
```

## 7. 发散-收敛法

### 发散阶段

- 规则：不评判、不打断、数量优先。
- 方法：头脑风暴、逆向思考、类比、换位思考。
- 目标：列出 5-10 个可能的方案。

### 收敛阶段

- 用目标匹配度、可操作性、成本、收益、风险筛选。
- 选出 1-3 个优先方案。

### 注意

- 不要边生成边评判。
- 如果只有 1 个方案，强迫自己再想出 2 个，避免二元选择。

## 9. 增量拆分法

### 适用场景

最终成果很难一开始想清楚的复杂任务，如写作、设计、策划、开发。

### 核心思想

先做出一个粗糙但完整的版本，再一轮一轮更新，而不是按固定顺序完成每个部分。

### 示例：写报告

```text
第一轮：骨架版
- 列出报告要回答的问题
- 分为几个部分
- 每个部分想说明什么
- 可能得出什么结论

第二轮：内容版
- 补充例子、解释、数据
- 理顺逻辑

第三轮：表达版
- 修改句子、统一风格、提升可读性

第四轮：细节版
- 检查格式、错别字、引用
```

### 关键提醒

- 第一版不需要好，只需要完整。
- 每一轮都增加价值。
- 如果完全不了解任务，先借鉴别人经验或做探索性尝试。

## 10. 0-10 标尺与下一步

### 标尺问句

```text
0 分 = 问题最糟糕的状态
10 分 = 问题被最理想地解决
现在大概是几分？
```

### 下一步设计

```text
提升 1 分会是什么样子？
做什么可以让分数提升 1 分？
这个行动什么时候开始？
```

### 使用规则

- 不要试图从 3 分直接跳到 10 分。
- 每次只追求 1 分进步，获取反馈后再继续。

## 11. 脚本示例

### 脚本 1：帮助用户聚焦一个问题

```text
用户：我工作不顺、感情不顺、身体也不好，一团糟。

教练：听起来你同时被几件事压着。我们今天先选一个最影响你、
或者解决了会让其他问题也减轻的“问题之王”。你会选哪一个？
```

### 脚本 2：澄清问题空间

```text
用户：我不知道要不要换工作。

教练：如果换工作，最理想的状态是什么样？如果不换，
你期待现状变成什么样？现在我们大概在哪个位置？
```

### 脚本 3：识别伪问题

```text
用户：我必须让所有人都认可我。

教练：这是一个你很在意的事情。但它是一个可以彻底解决的问题，
还是一个需要接受并重新框定目标的事情？比如，从“让所有人认可”变成“找到认同我的人”。
```

### 脚本 4：寻找例外

```text
用户：我和伴侣总是为家务吵架。

教练：有没有哪一次，你们没有为家务吵架？那次有什么不同？
可能是谁做了什么、环境怎样、或者当时情绪状态不一样？
```

### 脚本 5：设计最小实验

```text
用户：我想转行，但不知道新行业是否适合我。

教练：我们可以设计一个最小信息收集实验。比如，
约一位这个行业的人聊 30 分钟，或者先做一个相关的小项目。
你想先尝试哪一个？
```
