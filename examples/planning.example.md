# 计划示例

本示例是公开、虚构、脱敏案例，用来展示 `planning` 如何把目标拆成项目、任务、下一步行动、缓冲和复盘点。

## 用户输入

```text
我想接下来一个月开始稳定写公众号，但每次计划都很大，最后一篇也发不出来。帮我做一个现实一点的计划。
```

## 推荐路由

优先使用 `planning`。如果用户不知道为什么要写，先转 `life-vision`。如果用户卡在开始，合并 `procrastination-execution`。

## 计划思路

```text
我们先把“稳定写公众号”拆成一个月内可以验证的目标，而不是一上来设计长期内容帝国。

这个月的目标可以不是“成为稳定创作者”，而是：

在 4 周内发布 3 篇文章，并建立一个可以复用的选题、起稿、修改、发布流程。

这里有两个成功标准：

1. 结果标准：发出 3 篇。
2. 系统标准：知道自己从选题到发布最容易卡在哪里。
```

## 一个月计划

```text
第 1 周：建立最小系统

- 列 10 个候选选题。
- 选出 3 个最容易写的题。
- 给每个题写一句核心问题。
- 完成第 1 篇的粗纲。
- 周末发布或至少完成 80% 草稿。

第 2 周：发布第 1 篇，起草第 2 篇

- 发布第 1 篇。
- 记录写作过程卡点。
- 第 2 篇只追求清楚，不追求完整体系。
- 安排两个固定写作块，每次 60-90 分钟。

第 3 周：发布第 2 篇，优化流程

- 发布第 2 篇。
- 对比第 1 篇和第 2 篇：哪个环节更顺？
- 建立一个文章模板：问题、场景、方法、例子、结尾。
- 起草第 3 篇。

第 4 周：发布第 3 篇，复盘系统

- 发布第 3 篇。
- 复盘一个月：选题、起稿、修改、发布分别用了多久。
- 决定下个月保留什么、删掉什么、降低什么标准。
```

## 每周最低版本

```text
如果状态很好：
完成一篇 1500-2500 字文章。

如果状态一般：
完成一篇 800-1200 字短文。

如果状态很差：
只完成一个公开短笔记，保留发布动作。
```

## 复盘问题

```text
每周只问四个问题：

1. 我这周有没有发布或接近发布？
2. 最卡的是选题、起稿、修改，还是发布前焦虑？
3. 哪个动作让我更容易继续？
4. 下周只改一个变量，我要改什么？
```

## 回应收尾

```text
这个计划的重点不是一个月后证明你是稳定创作者，而是用 3 篇文章跑通一条真实流程。只要流程跑通，后面才有资格谈频率、定位和长期系列。
```

## 这个案例展示的能力

- 把愿望改成一个月可验证目标。
- 同时设置结果标准和系统标准。
- 设计高/中/低精力版本。
- 把计划和复盘连起来。
